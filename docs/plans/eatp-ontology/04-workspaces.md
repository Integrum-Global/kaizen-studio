## Overview

**Workspaces** are purpose-driven collections of work units that can cross departmental boundaries. Unlike traditional folder structures, workspaces are lenses that provide different organizational views of the same underlying work units.

### Archival

When a temporary workspace expires or is archived:

1. **Delegation records are revoked** - members lose access
2. **Audit trail is preserved** - all actions remain in history
3. **Work units are unaffected** - they stay in their original location
4. **Workspace becomes read-only** - can view history but not use

## Implementation Notes

### Database Schema

```sql
-- Workspaces table
CREATE TABLE workspaces (
    id UUID PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    workspace_type VARCHAR(20) NOT NULL, -- 'permanent', 'temporary', 'personal'
    owner_id UUID REFERENCES users(id),
    organization_id UUID REFERENCES organizations(id),
    expires_at TIMESTAMP,
    archived_at TIMESTAMP,
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- Workspace membership (work units)
CREATE TABLE workspace_work_units (
    workspace_id UUID REFERENCES workspaces(id),
    work_unit_id UUID REFERENCES work_units(id),
    delegation_id UUID REFERENCES delegations(id), -- EATP delegation record
    constraints JSONB, -- workspace-level constraint tightening
    added_at TIMESTAMP,
    added_by UUID REFERENCES users(id),
    PRIMARY KEY (workspace_id, work_unit_id)
);

-- Workspace membership (users)
CREATE TABLE workspace_members (
    workspace_id UUID REFERENCES workspaces(id),
    user_id UUID REFERENCES users(id),
    role VARCHAR(20) NOT NULL, -- 'owner', 'admin', 'member', 'viewer'
    constraints JSONB, -- per-member constraint tightening
    joined_at TIMESTAMP,
    invited_by UUID REFERENCES users(id),
    PRIMARY KEY (workspace_id, user_id)
);
```
